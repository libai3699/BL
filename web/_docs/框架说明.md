# 框架说明

## 关于一些架构包的说明

### eslint

主要用于代码校验，能够在开发阶段发现很多潜在问题

配置文件在根目录: .eslintrc.json
相关npm包:
yarn add eslint -D

### stylelint

主要用于对css样式进行代码格式化

配置文件在根目录: .stylelintrc.json
相关npm包:
如果用less
yarn add stylelint stylelint-less stylelint-config-standard stylelint-config-recommended-less postcss-less -D
如果用sass
yarn add stylelint stylelint-config-standard-scss -D
less配置
{
"extends": [
"stylelint-config-standard",
"stylelint-config-recommended-less"
],
"plugins": ["stylelint-less"],
"overrides": [
{
"files": ["**/*.less"],
"customSyntax": "postcss-less"
}
],
"rules": {
"selector-class-pattern": "^[a-z][a-zA-Z0-9]+$"
  }
}
sass配置
{
  "extends": "stylelint-config-standard-scss",
  "rules": {
    "selector-class-pattern": "^[a-z][a-zA-z0-9]+$"
}
}

### prettier

主要用于代码格式化，可以让大家的代码风格趋于统一，方便后续维护迭代
eslint-config-prettier是用来让eslint继承prettier的提示

配置文件在根目录: .prettierrc
相关npm包:
yarn add prettier eslint-config-prettier -D

### 关于husky lint-staged commitlint

husky是git勾⼦，在git操作时能执⾏⼀些命令
这样可以实现:
提交前使⽤lint-staged检测暂存区的⽂件，配合prettier规范代码。
提交时使⽤commitlint验证是否符合提交规范。

配置文件夹在根目录: .husky
commitlint配置文件在根目录: commitlint.config.js

其他配置处理:
在package.json文件中的scripts里添加命令："prepare": "husky install"
在package.json中添加lint-staged字段，内容如下：
JSON
"lint-staged": {
"\*.{js,css,md,ts,tsx,jsx}": "prettier --write"
}

相关npm包:
yarn add husky lint-staged commitlint @commitlint/config-conventional @commitlint/cli -D
